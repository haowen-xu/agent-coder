# 需求讨论

本目录用于沉淀需求讨论、范围确认与版本边界。

## 文件约定

- 建议文件名：`YYYY-MM-DD-<topic>.md`
- 每个需求文档至少包含：
  - 背景与目标
  - In Scope / Out of Scope
  - 用户角色与核心流程
  - 功能需求（Must/Should/Could）
  - 非功能需求（性能、稳定性、安全、可观测性）
  - 验收标准
