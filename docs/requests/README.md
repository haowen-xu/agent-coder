# 需求文档规范

本目录用于沉淀“仍有效的需求约束与定稿”。

## 文件约定

- 建议命名：`YYYY-MM-DD-<topic>.md`
- 文档至少包含：
  - 背景与目标
  - In Scope / Out of Scope
  - 关键流程或领域模型
  - 验收标准

## Issue 草稿约定

- Issue 草稿统一放仓库根目录 `issue-drafts/`（本地协作目录，不入库）。
- `docs/requests/` 不再存放临时 issue-draft 文档。

## 清理策略

- 过时讨论稿（`v1/v2/v3/...`）默认清理。
- 历史版本如需追溯，使用 Git 历史，不在目录中长期并列保留。
